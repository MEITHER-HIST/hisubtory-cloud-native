# SLO 및 가용성 명세서

## 1. 서비스 수준 목표 (SLO)
모니터링 대시보드(Prometheus/Grafana)를 통해 아래 지표를 준수하는지 상시 감시합니다.

| 지표명 | 목표 수치 | 측정 방법 |
| :--- | :--- | :--- |
| **응답 지연 시간** | 2초 이내 (p99) | Gateway 및 각 서비스의 HTTP Request Duration |
| **API 성공률** | 99.9% 이상 | 2xx, 3xx 응답 비율 / 전체 요청 수 |
| **로그 수집율** | 100% | Loki에 수집된 컨테이너 로그 정상 여부 |

## 2. 가용성 및 복구 기준
- **장애 인지:** Grafana Alerting을 통해 Slack/Email로 즉시 전송.
- **자동 복구(Auto-healing):** Docker Compose의 `restart: always` 정책을 통해 프로세스 중단 시 즉시 재시작.
- **최대 허용 장애 시간:** 월간 총 43분 이내 (99.9% 가용성 기준).

## 3. 리소스 사용량 임계치
- **CPU:** 80% 이상 지속 시 컨테이너 스케일 아웃 검토.
- **Memory:** 90% 이상 시 OOM(Out Of Memory) 방지를 위한 가비지 컬렉션 및 누수 점검.
